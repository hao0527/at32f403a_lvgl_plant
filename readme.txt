1.CMSIS / CORE（内核驱动程序）
arm内核
2.Startup(单片机启动程序)
汇编启动文件
3.Hardware / Midware(外部硬件驱动程序)
用户编写的硬件驱动，连接底层与应用层，承上启下；
类似分层架构，每个模块会提供接口，和上下层交互；
app层调用的代码最好是硬件无关代码；
4.Basic（内部功能的驱动程序）
这里ntc转换为温度和存储模块这两个可以理解为中间件；存储模块是硬件无关的，就属于basic。


封层好处：好移植，ntc换了我只改个ntc_user_config，单片机换了我改一个my_adc；
高复用，存储模块除了存温度还能存别的需要存的东西，可以通过存储模块的config来配置；
好维护；
